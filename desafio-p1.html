Um pouco de história

(voltar)

HTML (HyperText Markup Language) surgiu no início da década de 1990, criado por Tim Berners-Lee, o pai da World Wide Web.

A ideia era simples, mas revolucionária: permitir que documentos pudessem ser conectados por meio de links, formando uma rede de informações acessível de qualquer lugar do mundo.

Essa linguagem não é de programação, mas sim de marcação, responsável por dar estrutura e significado às páginas da web. Em outras palavras, é como o esqueleto de um site.

No HTML, podemos escrever elementos como h1 até h6 para definir títulos, e usar marcas como sup em "WWWTM" para indicar símbolos ou observações acima do texto.

“A Web não apenas conecta máquinas, ela conecta pessoas.”
— Tim Berners-Lee

Estrutura básica de uma página

(voltar)

Toda página HTML segue uma base comum. A tag <html> envolve todo o conteúdo, e dentro dela encontramos duas partes principais:

  • head → onde ficam informações invisíveis para o usuário, como o título da aba, links para estilos e metadados.
  • body → a parte visível, com textos, imagens, botões e todo o conteúdo exibido na tela.

Essa divisão entre “cabeça” e “corpo” ajuda a organizar o documento e a separar configurações do conteúdo em si.

Trabalhando com textos

(voltar)

Textos são a base da maioria dos sites, e o HTML oferece diversas tags para formatá-los:

  • Parágrafos → p define blocos de texto.
  • Títulos → de h1 (mais importante) a h6 (menos importante), organizam o conteúdo hierarquicamente.
  • Ênfase → strong para destacar palavras com importância, e em para ênfase, geralmente exibido em itálico.
  • Quebra de linha → br insere uma quebra de linha sem iniciar um novo parágrafo.
  • Comentários → !-- comentário -- permite adicionar notas no código que não aparecem na página.

Imagens e links

(voltar)

Imagens e links são essenciais para enriquecer a experiência do usuário:

  • Imagens → a tag img insere imagens, com atributos como src (caminho do arquivo) e alt (texto alternativo para acessibilidade).
  • Links → a tag a cria hiperlinks, permitindo navegar entre páginas ou sites. O atributo href define o destino do link.

Esses elementos tornam as páginas mais visuais e interativas, facilitando a navegação e o entendimento do conteúdo.

Listas

(voltar)

Quando precisamos listar informações, temos duas opções principais:

  • Listas ordenadas (ol) : numeradas, ideais para sequências ou rankings.
  • Listas não ordenadas (ul) : com marcadores, boas para tópicos soltos.
  • Itens de lista (li) : cada elemento individual da lista.

Formulários

(voltar)

Formulários são cruciais para interatividade, permitindo que usuários enviem dados:

  • Form (form) → envolve todo o formulário e define o método de envio (GET ou POST) e o destino dos dados.
  • Input (input) → cria campos de entrada variados, como texto, senha, email, checkbox, radio, entre outros.
  • Label (label) → associa um texto descritivo a um campo de entrada, melhorando a acessibilidade.
  • Textarea (textarea) → cria uma área maior para textos longos.
  • Select (select) e Option (option) → permitem criar listas suspensas para seleção de opções.
  • Button (button) → cria botões para enviar o formulário ou executar ações específicas.

Essas tags facilitam a coleta de informações dos usuários, essenciais para cadastros, pesquisas e interações diversas.

Estruturação e layout

(voltar)

Para organizar o conteúdo e criar layouts mais complexos, o HTML5 introduziu várias tags semânticas:

  • Header (header) → define o cabeçalho da página ou de uma seção.
  • Nav (nav) → agrupa links de navegação.
  • Main (main) → indica o conteúdo principal da página.
  • Section (section) → delimita seções temáticas do conteúdo.
  • Article (article) → representa um conteúdo independente, como um post de blog.
  • Aside (aside) → contém informações complementares, como barras laterais.
  • Footer (footer) → define o rodapé da página ou de uma seção.

Essas tags ajudam a estruturar o documento de forma clara e acessível, facilitando tanto a leitura humana quanto a interpretação por máquinas, como motores de busca.

Mídias integradas

(voltar)

Com HTML5, ficou simples adicionar sons e vídeos direto na página:

  • Audio (audio) → toca arquivos de som, com suporte a controles básicos.
  • Video (video) → exibe vídeos, podendo incluir capa, legendas e diferentes formatos.
  • Track (track) → adiciona legendas e descrições para acessibilidade.

Além disso, temos o Iframe (iframe), que permite incorporar conteúdos externos como vídeos do YouTube e mapas do Google.

Semântica e acessibilidade

(voltar)

Usar tags semânticas não é só uma questão de organização, mas também de acessibilidade:

  • Div (div) → cria blocos genéricos para agrupar conteúdo, mas sem significado específico.
  • Span (span) → similar ao div, mas para elementos inline, útil para estilização de partes do texto.
  • Attributes (atributos) → como id, class, title, entre outros, ajudam a identificar e descrever elementos, facilitando o trabalho de leitores de tela e motores de busca.

Adotar boas práticas semânticas melhora a experiência para todos os usuários, incluindo aqueles com deficiências.

Tabelas

(voltar)

Tabelas são úteis para organizar dados em linhas e colunas:

Tag Função
table Cria a tabela.
tr Define linhas.
td Células de dados.
th Células de cabeçalho.
thead, tbody, tfoot Ajudam na organização semântica.
caption Adiciona uma legenda explica

Conclusão

(voltar)

O HTML é o alicerce de qualquer site. Ele nasceu com o propósito de conectar documentos e evoluiu para suportar texto, mídia, interatividade e semântica, sempre com foco em **acessibilidade e padronização.

Ao dominar suas principais tags, você ganha a base necessária para construir páginas bem estruturadas, preparadas para receber estilo com CSS e lógica com JavaScript.
